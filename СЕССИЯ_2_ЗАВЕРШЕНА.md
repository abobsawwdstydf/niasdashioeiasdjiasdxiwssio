## 🎯 Реализованные функции

### 1. ✅ Кнопка "Спросить AI об этом сообщении"

**Где:** Контекстное меню сообщения (MessageBubble.tsx)

**Как работает:**
- Правый клик на сообщение → "Спросить AI"
- Открывает страницу Nexo AI с предзаполненным контекстом
- Передаёт messageId, chatId и текст сообщения через URL параметры
- AI автоматически загружает последние 10 сообщений из чата для контекста

**Файлы:**
- `apps/server/web/src/components/MessageBubble.tsx` - добавлена кнопка в контекстное меню
- `apps/server/web/src/pages/NexoAIPage.tsx` - обработка URL параметров
- `apps/server/src/routes/ai.ts` - endpoint `POST /api/ai/context`

**API Endpoint:**
```typescript
POST /api/ai/context
Body: { messageId, chatId, question? }
Response: { text: string, context: string }
```

### 2. ✅ Умные предложения ответов

**Где:** Над полем ввода сообщения (MessageInput.tsx)

**Как работает:**
- При получении нового сообщения от собеседника автоматически загружаются 3 варианта ответа
- AI анализирует последнее сообщение и предлагает:
  1. Формальный ответ
  2. Дружеский ответ
  3. Эмоциональный ответ
- Клик на предложение → текст вставляется в поле ввода
- Можно закрыть предложения кнопкой X

**Файлы:**
- `apps/server/web/src/components/MessageInput.tsx` - UI и логика
- `apps/server/src/routes/ai.ts` - endpoint `POST /api/ai/suggestions`
- `apps/server/web/src/lib/api.ts` - метод `getAISuggestions()`

**API Endpoint:**
```typescript
POST /api/ai/suggestions
Body: { chatId, lastMessage }
Response: { suggestions: string[] }
```

**Особенности:**
- Загружаются только для сообщений от других пользователей (не от себя)
- Минимальная длина сообщения: 3 символа
- Автоматическая загрузка через WebSocket событие `new_message`

### 4. ✅ Исправление грамматики

**Где:** Кнопка в поле ввода (MessageInput.tsx)

**Как работает:**
- Кнопка с иконкой документа появляется когда есть текст
- Клик → открывается модальное окно с проверкой
- AI анализирует текст и показывает:
  - Исходный текст (красная рамка)
  - Исправленный текст (зелёная рамка)
  - Кнопка "Применить исправления"
- Если ошибок нет → показывается галочка

**Файлы:**
- `apps/server/web/src/components/MessageInput.tsx` - UI и логика
- `apps/server/src/routes/ai.ts` - endpoint `POST /api/ai/grammar`
- `apps/server/web/src/lib/api.ts` - метод `checkGrammar()`

**API Endpoint:**
```typescript
POST /api/ai/grammar
Body: { text }
Response: { 
  corrected: string, 
  hasChanges: boolean, 
  original: string 
}
```

## 🔧 Технические детали

### Backend (apps/server/src/routes/ai.ts)

Добавлены 4 новых endpoint'а:
1. `POST /api/ai/context` - AI с контекстом из сообщения
2. `POST /api/ai/suggestions` - Умные предложения ответов
3. `POST /api/ai/autocomplete` - Автодополнение текста
4. `POST /api/ai/grammar` - Проверка грамматики

Все endpoint'ы используют существующую систему балансировки AI провайдеров:
- Cerebras (приоритет)
- Groq (резерв)
- SambaNova (глубокий резерв)
- OpenRouter (последний рубеж)

### Frontend

**Новые компоненты:**
- Модальное окно проверки грамматики (в MessageInput.tsx)
- Панель умных предложений (в MessageInput.tsx)
- Кнопка "Спросить AI" в контекстном меню (в MessageBubble.tsx)

**Новые методы API (apps/server/web/src/lib/api.ts):**
```typescript
async getAIContext(messageId, chatId, question?)
async getAISuggestions(chatId, lastMessage)
async getAIAutocomplete(text)
async checkGrammar(text)
async post(endpoint, data) // Универсальный метод
async get(endpoint) // Универсальный метод
```
