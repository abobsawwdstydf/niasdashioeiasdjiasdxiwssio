## 📋 Выполненные задачи

### 1. ✅ Backend API для секретных чатов
**Файл**: `apps/server/src/routes/secretChats.ts`

**Endpoints**:
- `POST /api/secret-chats/create` - Создание секретного чата
  - Параметры: userId, password (опционально), selfDestructTimer (опционально)
  - Автоматически включает E2E шифрование
  - Хеширует пароль с помощью bcrypt
  
- `POST /api/secret-chats/verify` - Проверка пароля секретного чата
  - Параметры: chatId, password
  - Возвращает success: true/false
  
- `POST /api/secret-chats/message/self-destruct` - Установка таймера самоуничтожения
  - Параметры: messageId, timer (в секундах)
  - Вычисляет время удаления (selfDestructAt)
  
- `DELETE /api/secret-chats/:chatId` - Удаление секретного чата
  - Удаляет все сообщения и сам чат
  
- `GET /api/secret-chats/:chatId/settings` - Получение настроек секретного чата
  - Возвращает: isSecret, isE2E, hasPassword
  
- `POST /api/secret-chats/:chatId/screenshot` - Уведомление о попытке скриншота
  - Отправляет уведомление другому участнику

**Безопасность**:
- Все endpoints защищены middleware `authenticateToken`
- Пароли хешируются с помощью bcrypt (10 раундов)
- Проверка прав доступа (только участники чата)

## 🎨 Как использовать

### Создание секретного чата:
1. Открыть профиль пользователя
2. Нажать кнопку "Секретный чат" (с градиентом)
3. Настроить параметры:
   - Установить пароль (опционально)
   - Выбрать таймер самоуничтожения
4. Нажать "Создать"
5. Чат автоматически откроется

### Индикация секретного чата:
- В списке чатов: иконка замка 🔒 рядом с именем
- В заголовке чата: иконка замка 🔒 слева от имени
- Цвет иконки: фиолетовый (`text-purple-400`)

### Возможности секретного чата:
- **E2E шифрование**: Автоматически включается при создании
- **Самоуничтожающиеся сообщения**: Настраивается при создании чата
- **Защита от скриншотов**: Уведомление собеседнику при попытке скриншота
- **Пароль**: Опциональная защита паролем

## 📦 Технические детали

### Безопасность
- Пароли хешируются с помощью bcrypt (10 раундов)
- E2E шифрование включается автоматически (`isE2E: true`)
- Проверка прав доступа на всех endpoints
- Защита от CSRF через JWT токены

### База данных
Используются существующие поля в схеме Prisma:
- `Chat.isSecret` - флаг секретного чата
- `Chat.isE2E` - флаг E2E шифрования
- `Chat.secretPassword` - хешированный пароль
- `Message.selfDestructTimer` - таймер в секундах
- `Message.selfDestructAt` - время удаления
- `Message.canForward` - можно ли пересылать
- `Message.canScreenshot` - можно ли делать скриншот

### Таймеры самоуничтожения
Доступные варианты:
- Без таймера (null)
- 5 секунд
- 10 секунд
- 30 секунд
- 1 минута (60 сек)
- 5 минут (300 сек)
- 1 час (3600 сек)
- 1 день (86400 сек)

## 🐛 Известные ограничения

### Не реализовано в этой сессии:
1. **Фактическое E2E шифрование сообщений** - Требует Signal Protocol
2. **Автоматическое удаление сообщений** - Требует фоновый процесс (уже есть в `selfDestructCleanup.ts`)
3. **Детекция скриншотов** - Невозможно в веб-браузере (только уведомление вручную)
4. **Запрет пересылки** - Требует проверку на фронтенде
5. **Проверка пароля при открытии чата** - Требует дополнительный UI

### Технические ограничения:
- Детекция скриншотов невозможна в веб-браузере
- E2E шифрование требует обмена ключами (Signal Protocol)
- Самоуничтожение работает только на сервере (клиент может сохранить)
